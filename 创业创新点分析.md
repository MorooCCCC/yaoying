# Yao-Vision（爻影）创业创新点分析文档

> 基于 Next.js + AI + 手势识别的六爻占卜应用

---

## 一、项目概述

### 1.1 核心功能

| 阶段 | 功能描述 | 技术亮点 |
|------|----------|----------|
| **开屏** | 八卦环旋转 + 阴阳鱼动画 + 浮动粒子背景 | Framer Motion 动画，鼠标跟随光晕 |
| **用户画像** | 收集年龄、性别、所在地、问题、时限 | 数据驱动解卦，个性化报告 |
| **摇卦** | 手势识别摇卦 / 手动按钮摇卦 | MediaPipe Hands 实时手部追踪 |
| **排盘** | 本卦/变卦可视化 + 干支四柱 | 自实现干支历法算法，64卦完整数据库 |
| **AI 解卦** | GPT-4o 流式生成专业解卦报告 | Vercel Edge Runtime，流式输出 |

### 1.2 技术栈

```
┌──────────────────────────────────────────────────────────┐
│                      前端 / 全栈                          │
│  Next.js 14 (App Router) + TypeScript + Tailwind CSS     │
├───────────────┬──────────────────────────────────────────┤
│  动画引擎      │  Framer Motion（页面切换/铜钱翻转/粒子）   │
│  音效引擎      │  Web Audio API（纯代码合成，无音频文件）    │
│  手势识别      │  MediaPipe Hands（CDN动态加载）            │
│  历法计算      │  自实现干支算法（lunar-javascript备用）     │
│  AI 接入       │  Vercel AI SDK + @ai-sdk/openai           │
│  AI 模型       │  GPT-4o（Edge Runtime流式输出）            │
│  部署          │  Vercel（Edge Function + 全球CDN）         │
└───────────────┴──────────────────────────────────────────┘
```

### 1.3 音效系统（核心亮点）

整套音效**零依赖音频文件**，全用 Web Audio API 实时合成：

| 触发场景 | 合成原理 |
|---------|---------|
| 摇晃铜钱 | 白噪声 burst + 高通滤波 + 随机金属泛音 |
| 铜钱落定 | 正弦衰减 + 非整数倍泛音 + 短混响（模拟青铜） |
| 一爻成卦 | 五声音阶单音 + 轻微颤音 + 混响 |
| 完卦 | C宫调四音和弦依次入 + 底鼓（庄重感） |
| AI 解卦启动 | 五声音阶上行 + 高八度回声（水晶碗效果） |
| 页面切换 | 低通滤波白噪声（气流感） |

### 1.4 成本分析

| 项目 | 费用 |
|------|------|
| 单次解卦（输入~800 token，输出~1000 token） | ~$0.012 ≈ ¥0.085 |
| 推荐方案：Vercel 免费版 + gpt-4o | 月花费 ≤ ¥20 |
| 适用场景 | 自用 + 小圈子 |

---

### 2.2 Web Audio API 音效引擎（技术产品化）

**模式**: 将 `audio.ts` 封装成 npm 包，提供「零文件音效解决方案」

**产品名称建议**: `SoundCraft` / `ZeroAudio` / `WebSynth`

**核心卖点**:
- 零音频文件，纯代码合成
- 体积小（< 10KB），加载快
- 无版权风险
- 高度可定制

**目标用户**:
- 小游戏开发者（微信小游戏、H5游戏）
- 轻量级 Web 应用开发者
- 营销页/活动页开发者

**竞品对比**:
| 特性 | 传统音频文件 | Web Audio 引擎 |
|------|-------------|---------------|
| 文件大小 | 100KB-数MB | < 10KB |
| 加载时间 | 需等待下载 | 即时生成 |
| 版权风险 | 有 | 无 |
| 定制性 | 低 | 高 |
| 网络依赖 | 高 | 低 |

---

### 2.3 手势识别交互 SDK

**模式**: 封装 MediaPipe + 自定义手势识别逻辑，提供开箱即用的手势交互能力

**产品名称建议**: `GestureKit` / `HandFlow`

**技术壁垒**:
- 移动端摄像头兼容性优化（已解决黑屏问题）
- 手势动作识别算法（摇晃、点击、滑动等）
- 性能优化（低延迟、低功耗）

**应用场景**:
| 场景 | 功能 | 示例 |
|------|------|------|
| 健身 App | 动作计数、姿势纠正 | 深蹲计数、瑜伽姿势检测 |
| 教育 App | 手势答题、互动教学 | 选择题手势选择 |
| 游戏 | 体感控制 | 手势操控角色 |
| 无障碍 | 手势操作辅助 | 视障用户手势导航 |

## 四、推荐策略

### 4.1 资源有限时（个人/小团队）

**优先方向**: AI 传统文化 SaaS + 情绪价值内容付费

**理由**:
- 轻资产运营
- 高毛利（AI 成本可控）
- 易传播（社交属性强）
- 可快速验证 MVP

**启动步骤**:
1. 基于现有 yao-vision 代码，封装成微信小程序
2. 接入微信支付，设置会员订阅
3. 抖音/小红书内容引流
4. 根据数据反馈迭代

### 4.2 有一定技术积累时

**优先方向**: Web Audio 音效引擎 + 手势识别 SDK

**理由**:
- 技术壁垒高，竞争少
- 可服务 B 端客户
- 技术资产可复用

**启动步骤**:
1. 将现有代码模块化、文档化
2. 发布开源版本获取用户反馈
3. 推出 Pro 版本（高级功能付费）
4. 拓展企业客户

### 4.3 设计/品牌能力强时

**优先方向**: 新中式 UI 组件库 + 品牌联名

**理由**:
- 差异化明显
- 可与其他产品联动
- 品牌溢价空间大

**启动步骤**:
1. 整理设计系统，发布组件库
2. 与文创品牌/IP 联名
3. 拓展博物馆/文化机构客户
4. 开发配套设计工具

## 五、风险提示

| 风险 | 说明 | 应对策略 |
|------|------|----------|
| 政策风险 | 占卜类内容可能受监管 | 定位为「文化娱乐」而非「迷信」，避免绝对化表述 |
| 内容风险 | AI 生成内容可能不当 | 设置内容审核机制，人工抽检 |

| 竞争风险 | 同类产品增多 | 强化差异化（手势交互、音效、UI） |
| 成本风险 | AI 调用成本上升 | 优化 Prompt，缓存常见结果，设置调用上限 |

### 6.2 核心算法说明

**铜钱摇卦逻辑**:
```
三枚铜钱 → 正面数统计 → 确定爻型
  3正 = 老阳（动爻，本卦阳→变阴）
  2正 = 少阴（静，本卦阴）
  1正 = 少阳（静，本卦阳）
  0正 = 老阴（动爻，本卦阴→变阳）
```

**干支历法计算**:
```
当前时间 → 年柱/月柱/日柱/时柱（天干地支）
→ 旬空（每旬末尾2支）
→ 月破（与月建相冲的地支）
```
